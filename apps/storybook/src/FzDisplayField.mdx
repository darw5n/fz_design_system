# FzDisplayField

Read-only key-value display component for presenting labeled data. Renders a label above a value with configurable text sizes, emphasis, and spacing.

## Features

- Label and value pair display in a vertical layout
- Configurable label size (small or normal)
- Emphasized (bold) value support
- Three gap options between label and value (none, small, medium)
- Uses design system typography tokens

## Installation

```bash
npm install @fiscozen/displayfield
```

## Basic Usage

```vue
<script setup>
import { FzDisplayField } from '@fiscozen/displayfield'
</script>

<template>
  <FzDisplayField label="Full Name" value="John Doe" />
</template>
```

## Examples

### Different Sizes

```vue
<template>
  <!-- Small label (14px) -->
  <FzDisplayField label="Email" value="john@example.com" size="small" />

  <!-- Normal label (16px, default) -->
  <FzDisplayField label="Email" value="john@example.com" size="normal" />
</template>
```

### Emphasized Value

```vue
<template>
  <!-- Normal weight -->
  <FzDisplayField label="Status" value="Active" />

  <!-- Bold/semibold value -->
  <FzDisplayField label="Status" value="Active" :isEmphasized="true" />
</template>
```

### Gap Options

```vue
<template>
  <!-- No gap (default) -->
  <FzDisplayField label="Label" value="Value" gap="none" />

  <!-- Small gap (8px) -->
  <FzDisplayField label="Label" value="Value" gap="small" />

  <!-- Medium gap (12px) -->
  <FzDisplayField label="Label" value="Value" gap="medium" />
</template>
```

### Complete Configuration

```vue
<template>
  <FzDisplayField
    label="Account Balance"
    value="EUR 12,500.00"
    size="small"
    :isEmphasized="true"
    gap="small"
  />
</template>
```

## Accessibility

- The component uses semantic HTML with `span` elements for label and value
- Label and value are visually distinguishable through color contrast (grey label, black value)
- Text is accessible to screen readers in natural reading order (label first, then value)

## Best Practices

- Use `FzDisplayField` for read-only data presentation, not for editable fields
- Choose `isEmphasized` when the value needs to stand out (e.g., totals, important statuses)
- Use `size="small"` for dense layouts or secondary information
- Pick an appropriate `gap` based on the surrounding layout density
